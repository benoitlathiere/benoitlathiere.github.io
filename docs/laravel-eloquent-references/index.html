Laravel Eloquent - Référence

Dans ce dossier, je vais passer en revue les méthodes et classes proposées par l'ORM (object-relational mapping) Eloquent, choisi par l'équipe du framework Laravel. Vous trouverez sur le web des benchmarks qui montrent que les requêtes générées par Query Builder (lui aussi dans Laravel) sont plus performantes, notamment sur les grosses bases de données (beaucoup de lignes et des jointures complexes) ; mais si votre besoin est plus modeste, Eloquent fera parfaitement le travail et le code sera surtout plus lisible et plus simple à écrire, ce qui diminue le risque d'erreur.

La plupart des méthodes présentées ici peuvent être chainées, c'est-à-dire qu'on peut les mettre bout-à-bout, car chacune renvoie une variable de type Query Builder.

Fonction de rappel

Certaines méthodes proposent d'utiliser une fonction de rappel (callback function) pour ajouter des méthodes à la requête. La fonction reçoît la requête en paramètre et doit retourner la requête, complétée des méthodes que vous avez ajoutées.

Ici la fonction regroupe 2 conditions, elle remplace des parenthèses pour le OR :

// mélange de conditions OR et AND:
// (user_id.. OR user_id..) AND role IN (...)
$current_user_id = 5;
$users = User::::where( function($query) use ($current_user_id) {
            return $query->where('user_id', $current_user_id)
                ->orWhereIn('user_id', Auth::user()->friends()->lists('id'));
        })
        ->whereIn('role_id', [1, 5, 7]);

// équivalent en fonction fléchée, plus courte:
$users = User::::where( fn($query) => {
            $query->where('user_id', $current_user_id)->orWhereIn('user_id', Auth::user()->friends()->lists('id'));
        })
        ->whereIn('role_id', [1, 5, 7]);

L'équivalent en SQL :

SELECT * FROM users where (user_id = 27 OR user_id IN (12, 18, 32, 243) ) AND role_id IN (1, 5, 7);

Les fonctions peuvent être imbriquées les unes dans les autres, mais attention à la complexité de la requête finale, ce n'est pas toujours facile à débugguer !

Sélectionner les données souhaitées​

Je vais d'abord décrire les méthodes qui permettent d'extaire les informations (select statement) de la base de données.

select() / addSelect() / selectRaw()​

Ces 3 méthodes indiquent à Eloquent les colonnes que l'on veut récupérer (query statement).

select() et addSelect() nécessitent que l'on passe le nom des colonnes souhaitées en paramètres ; une liste de noms séparés par des virgules, ou dans un tableau. La deuxième est à utiliser si l'on a déjà appelé la première, afin d'ajouter des colonnes non prévues.

// on précise les colonnes attendues, sinon nous les auront toutes:
$users =  User::select('id', 'first_name', 'last_name')->get();
// ici on reçoit une liste possible des colonnes par une variable (whereHas() est vu plus loin):
$columns = ['age','date'];  // reçu en paramètre d'une méthode
$users = User::select('id', 'first_name', 'last_name')->whereHas(count($colums)>0, function ($query) use ($columns) {
    return $query->addSelect($columns); // on ajoute les colonnes
})->get();

selectRaw() permet de définir une colonne en langage SQL brut, ce qui peut être ^ratique si l'on veut utiliser une syntaxe impossible à écrire avec les méthodes de Query Builder. C'est une alternative à l'utilisation de DB::raw().

// permet d'ajouter une colonne indiquant si l'utilisateur est majeur :
$users = User::select('id', 'first_name', 'last_name')->selectRaw('CASE age>18 THEN 1 ELSE 0 END AS _is_adult')->get();

with()​

Cette méthode va vous permettre d'inclure des relations en mode "eager loading", c'est-à-dire charger la relation au plus tôt, en même temps que la requête.

💡 Cette technique charge en mémoire l'ensemble des lignes de la relation en une seule requête, plutôt que d'appeler la relation à chaque appel du modèle ("lazy loading"), donc avec une requête à chaque fois. Ce problème de chargement tardif est appelé "N+1", car il nécessite N+1 requêtes : la requête initiale + une requête à chaque appel de la relation. En revanche, vous chargez les éléments de la relation à la demande,au fur et à mesure, sans encombrer la mémoire.

😊 Le "eager loading" va ne faire que 2 requête : la requête du modèle principal, et la requête de la relation (qui récupère toutes les occurenences nécessaires). A contrario, quelle technique suppose que Laravel va charger en mémoire toutes les occurences, même si vous les utilisez pas nécessairement toutes. Avec de nombreuses relations importantes, vous risquez saturez la mémoire allouée à PHP.

La relation qui peut être chargée doit être déclarée dans la classe du modèle (voir annexe).

$users = User::select('id', 'first_name', 'last_name')->with('posts')->get();
foreach($users as $user) {
    $user->post->id;    // la propriété est au singulier, même si la relation est au pluriel.
}

withCount() / loadCount()​

La méthode fait la somme du nombre lignes d'une relation (voir annexe) liée à la requête principale. Elle combine à la fois with() d'Eloquent et count() de SQL. La méthode créé une propriété qui est la concaténation du nom de la relation et du mot '_count'. La relation, comme souvent avec Eloquent, est une sous-class de Builder. La méthode accepte plusieurs relations, un callback, et même un alias.

// on compte le nombre de posts pour chaque utilisateur :
$users = User::select('id', 'first_name', 'last_name')->withCount('posts')->get();
// accès à la propriété :
$users->first()->posts_count;
// on peut mettre plusieurs relations utiliser un callback, et mettre un alias :
$users = User::select('id', 'first_name', 'last_name')->withCount(['posts', 'friends as nb_adults' => function (Builder $query) {
    $query->where('age', '>=', 18);
}])->get();
$users[0]->posts_count;
$users[0]->nb_adults;

⚠️ Si vous utiliser select() dans la requête, il faut appeler withCount() après select().

Vous pouvez utiliser loadCount() sur la collection qui résulte de la requête, avec les mêmes possibilités (relations, callback, alias).

$users = User::select('id', 'first_name', 'last_name')->get();
$users->loadCount(['posts', 'friends as nb_adults' => function (Builder $query) {
    $query->where('age', '>=', 18);
}]);

Filtrer les données​

Maintenant on va voir comment filtrer les données, c'est-à-dire comment ecarter celles qui ne nous intéressent pas.

where() / whereIn() / whereNotIn()​

Toutes les méthodes commençant par where* permettent d'indiquer des conditions pour filtrer (conserver) les données correspondant aux critères spécifiés. Ces méthodes ont plusieurs signatures et les paramètres passés sont variables. Plusieurs possibiltiés :

  1. 2 paramètres, la colonne et la valeur qui doit être identique (comparaison d'égalité).
  2. 3 paramètres, le 2ème étant l'opérateur de comparaison (voir annexe).
  3. 2 paramètres, avec une fonction de rappel.
// 1. comparaison d'égalité implicite:
$users = User::select('id', 'first_name', 'last_name')->where('age', 18)->get();

// 2. comparaison explicite:
$users = User::select('id', 'first_name', 'last_name')->where('age', '>=', 18)->get();

whereIn(), whereNotIn() vérifient que la valeur d'une colonne est ou n'est pas présente dans un tableau.

$users = User::select('id', 'first_name', 'last_name')->where('status', ['active', 'idle','standby'])->get();
$users = User::select('id', 'first_name', 'last_name')->whereNotIn('prefered_language', ['Cobol', 'C','Ada'])->get();

Également disponibles :

  • orWhere()
  • orWhereIn()
  • orWhereNotIn()

whereBetween() / whereNotBetween()​

whereBetween(), orWhereBetween(), whereNotBetween(), orWhereNotBetween() vérifient ou non une valeur numérique entre 2 bornes incluses.

// l'âge entre 18 et 100:
$users = User::select('id', 'first_name', 'last_name')->whereBetween('age', [18, 100])->get();
// l'âge hors 18 à 100:
$users = User::select('id', 'first_name', 'last_name')->whereNotBetween('age', [18, 100])->get();

Également disponibles :

  • orWhereBetween()
  • orWhereNotBetween()

whereHas()​

Cette méthode vérifie la présence d'une ou plusieurs relations avec le modèle interrogé. Par exemple, si votre modèle User est lié à des Article, vous pouvez vouloir n'avoir que les User qui possèdent des Articles, ou l'inverse.

$user = User::whereHas('articles')->get();

whereDate() / whereYear() / whereMonth() / whereDay() / whereTime()​

whereDate() fait une comparaison sur une colonne contenant au moins une date (et même si la colonne est de type datetime ou timestamp). Le temps (heure, minute, seconde..) sont ignorés dans la comparaison.

whereDay() et whereMonth() font une comparaison sur uniquement, respectivement, sur le jour ou le mois d'un champ contenant au moins une date. De la même manière, whererYear() ne compare que l'année elle-même, sans tenir compte du reste.

Enfin, whereTime() ne compare que le temps (heure, mniute, seconde) selon le format PHP H:i:s.

Ces méthodes vous évite de faire appel aux méthodes SQL d'extraction des différentes parties d'une date.

Obtenir les résultats​

get()​

Cette méthode permet tout simplement d'obtenir le résultat de la requête. get() génère une Collection du modèle interrogé (compatible avec Illuminate\Support\Collection.

count()​

Au lieu de récupérer les données avec get() puis de compter les lignes, il est possibles d'appeler la méthode count() directement en fin de requête Builder.

exists() / doesntExist()​

Ces méthodes appelées en fin de Builder renvoient un booléen indiquant si au moins 1 occurence correspond à la recherche. Ces méthodes évitent de renvoyer des lignes qui n'auraient d'intérêt que pour le comptage.

// vérifier l'existence de lignes :
$rowsExist = $user = User::whereHas('articles')->count() > 0;
//devient :
$rowsExist = $user = User::whereHas('articles')->exists();
// vérifier l'absence' de lignes :
$rowsDoesntExist = $user = User::whereHas('articles')->count() == 0;
// devient :
$rowsDoesntExist = $user = User::whereHas('articles')->doesntExist();

Déclaration d'une relation dans une classe de Model​

Plusieurs méthodes font appel à une relation du modèle.

// déclaraton de la relation :
class User extends Model {
    // relation one-to-one :
    public function entity(): HasOne
    {
        return $this->hasOne(Entity::class);
    }
    // relation One-to-many :
    public function posts(): HasMany
    {
        return $this->hasMany(Post::class);
    }
}
